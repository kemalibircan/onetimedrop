## Three Methods, One Goal

When you need to move a file from one device to another quickly, you have three common wireless options: scan a QR code, share a link, or use Bluetooth. Each has real strengths — and real weaknesses.

## QR Code Transfer

### How it works
A QR code encodes a session URL. Scan it with your phone's camera → the join page opens automatically → upload files.

### Pros
- No typing required — one scan and you're in
- Works across different operating systems
- Fast: typical setup takes 5–10 seconds
- No prior pairing needed

### Cons
- Requires a camera on the phone
- Both devices need internet access
- Not peer-to-peer (files travel via server)

**Best for:** Transferring files to a computer you don't own, printing documents, quick multi-file transfers.

## Shared Link Transfer

### How it works
You upload a file to a cloud service (WeTransfer, Google Drive) and share a download link. The recipient opens the link and downloads.

### Pros
- Easy to share by copy-pasting
- Works asynchronously (recipient doesn't need to be online simultaneously)
- Supports very large files (Google Drive up to 5TB)

### Cons
- Files stay on the server indefinitely (often)
- Requires an account on most services
- Links can be forwarded to unintended recipients

**Best for:** Sending large files remotely to someone who needs it later.

## Bluetooth File Transfer

### How it works
Bluetooth creates a direct device-to-device connection over short range (~10m) and transfers files directly.

### Pros
- No internet required
- Completely local, doesn't go through any server
- Good for quick local sharing between paired devices

### Cons
- Must pair devices first (and pairing between different OSes is painful)
- Very slow for large files (Bluetooth 4.x: ~2–3 Mbps)
- Range limitation (~10 metres)
- Doesn't work across different ecosystems (iOS won't share files via Bluetooth with Android)

**Best for:** Sharing single small files between already-paired devices on the same physical location.

## Head-to-Head Comparison

| Factor | QR Code | Shared Link | Bluetooth |
|---|---|---|---|
| Setup time | 5–10s | 30–60s | 1–2 min |
| Requires account | ❌ | Usually ✅ | ❌ |
| Works across OS | ✅ | ✅ | Partial |
| Speed | Fast | Moderate | Slow |
| Privacy | High (auto-delete) | Low | High |
| Internet needed | ✅ | ✅ | ❌ |
| File size limit | 50MB (OneTimeDrop) | 2GB+ | Unlimited |

## The Verdict

For **casual, one-time, cross-device file transfer** (especially phone to unknown computer), QR code wins on speed and ease of setup.

For **large files sent remotely to someone asynchronously**, a shared link via cloud storage is best.

For **offline local sharing between already-paired devices**, Bluetooth works fine for small files.
