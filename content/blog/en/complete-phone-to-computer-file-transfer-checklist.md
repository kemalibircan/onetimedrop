## Why a Checklist Helps

Most file transfer failures are predictable: the file is in the wrong format, the method is chosen too late, the computer is shared, or the sender uses a tool that compresses media. A checklist prevents those mistakes before the clock starts.

Use this before any important phone-to-computer handoff.

## The Checklist

### 1. Identify the real file
Make sure you have the final version, not a screenshot, draft, or forwarded copy.

### 2. Pick the right format
For documents, prefer PDF. For editing, keep the original photo or video file.

### 3. Match the method to the moment
Use [OneTimeDrop](/) for nearby transfer, expiring links for remote delivery, and cable or USB for true offline fallback.

### 4. Think about the device
If the receiving computer is borrowed or public, avoid logging into broad personal accounts.

### 5. Confirm quality after download
Check file size, open the file, and verify the pages or media playback.

### 6. Clean up
Delete leftover downloads on shared machines and close the transfer session.

## Quick Reference Table

| Situation | Best first choice |
|---|---|
| Nearby laptop | Session-based browser transfer |
| Remote recipient | Expiring link |
| Large offline batch | Cable or USB |
| Sensitive shared-computer task | Temporary session plus cleanup |

## FAQ

**Q: What is the most common mistake?**  
A: Using the most familiar method instead of the most appropriate one.

**Q: Should I always avoid email?**  
A: No, but avoid it for large, sensitive, or urgent files when better options exist.

**Q: Is this checklist only for work?**  
A: No. It helps with travel, school, printing, meetings, and family documents too.
